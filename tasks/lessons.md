# Lessons

- Keep documentation examples generic and neutral (prefer `google.com` for URL examples).
- Preserve exact filename casing in docs/tests (`CLAUDE.md`, not `claude.MD`).
- Never add user-specific local dictionary entries to versioned files; keep them in ignored `config.toml`.
- In PowerShell sessions where Rust tools are not on `PATH`, use explicit `~/.cargo/bin/cargo.exe` for Rust checks.
